# 项目定位

## 项目是什么

Scrcpy Mobile 是一个运行在 Android 设备上的远程控制项目，目标是在 Android 设备之间完成：

- 屏幕镜像
- 控制输入
- 音视频传输
- 会话配置管理

它以 scrcpy 协议链路为基础，但不是简单照搬桌面端结构，而是将 ADB、scrcpy、媒体、会话和 UI 一起重新组织为 Android 本地应用。

## 项目要解决的问题

这个项目要解决的核心问题不是“如何显示远端画面”这么单一，而是要同时处理：

- 设备发现与连接
- ADB 建链
- scrcpy server 启动
- socket 通信
- 视频与音频处理
- 控制指令回传
- 会话配置保存
- UI 与设置入口

这些能力共同组成远控主链路。

## 当前技术形态

项目主要由以下技术栈构成：

- Kotlin
- Jetpack Compose
- MVVM
- C/C++ with CMake
- Android 平台能力

它不是纯 UI 应用，也不是单纯的 native 项目，而是一个由多层技术共同驱动的工程。

## 读者应该先建立什么认知

理解这个项目时，最重要的不是先记住所有类名，而是先建立三个认知。

### 第一，项目的核心边界是会话

设备配置、连接过程、运行状态、解码和控制，最终都要归到会话边界上理解。

### 第二，项目的核心链路是 ADB 到 scrcpy

从建立连接到拉起 server，再到 socket、metadata、decoder 和控制流，这条链路是最重要的主线。

### 第三，项目的核心难点是收敛复杂度

项目当前真正难的部分，不是少一个功能，而是：

- 避免事实源重复
- 避免全局入口泛滥
- 避免目录结构持续打散主链路

## 一句话总结

Scrcpy Mobile 是一个以会话和远控链路为核心的 Android 工程，理解它的关键不在于逐个看功能点，而在于顺着主链路理解配置、连接、运行时和工程约束。
